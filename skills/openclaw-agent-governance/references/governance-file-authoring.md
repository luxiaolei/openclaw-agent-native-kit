# Governance file authoring

## File roles

- `IDENTITY.md`: short identity card
- `SOUL.md`: persona, boundaries, core stance
- `AGENTS.md`: executable operating rules and delegation behavior
- `USER.md`: operator profile and preferences
- `TOOLS.md`: environment-specific notes
- `HEARTBEAT.md`: tiny periodic checklist only when truly needed
- `BOOTSTRAP.md`: first-run orientation, removable later
- `MEMORY.md`: durable specialist memory only

## Writing rules

- Put cross-cutting safety and delegation rules in `AGENTS.md`.
- Keep `SOUL.md` for tone, identity, and boundaries.
- Keep files short enough to be reliably useful, but concrete enough to govern behavior.
- Avoid vague corporate language that cannot change runtime decisions.
